## 2024-05-22 - AuthInput Accessibility\n**Aprendizado:** O componente `AuthInput` requer que um `id` seja passado via props para associar corretamente o label ao input (via `htmlFor`). Além disso, botões interativos dentro de inputs (como toggle de senha) não devem ter `tabIndex={-1}` para garantir acesso via teclado.\n**Ação:** Ao usar ou criar inputs customizados, sempre verificar a associação label-input e a navegabilidade via teclado de elementos auxiliares.\n\n## 2026-02-12 - Navegação por teclado em tabs de autenticação\n**Aprendizado:** Em navegação tipo "pills" para trocar telas, tratar o grupo como `tablist` com `aria-selected`, `tabIndex` em roving e atalhos `ArrowLeft/ArrowRight/Home/End` melhora bastante a usabilidade sem alterar layout.\n**Ação:** Reutilizar esse padrão em qualquer seletor horizontal de telas/etapas para manter experiência consistente entre mouse, teclado e leitor de tela.\n\n## 2024-05-25 - Melhoria de acessibilidade no ThemeToggle\n**Aprendizado:** Ao implementar botões apenas com ícones sem texto na tela, é fundamental adicionar `aria-label` e envolver num `Tooltip` usando o Shadcn para comunicar visualmente a função. Adicionar ring styles (`focus-visible:ring-2`) melhora a visualização do foco usando teclado.\n**Ação:** Sempre usar `Tooltip` com `aria-label` em todos os botões que não tem texto.
